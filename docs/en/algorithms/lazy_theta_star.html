Lazy Theta* (deferred line-of-sight)¶

Background¶

Theta*2 straightens the grid path by asking, on every relaxation, whether the current node's parent can see a candidate successor (line_of_sight). That LOS query is the expensive part of the algorithm, and Theta* pays it once per edge — most of which are later discarded.

Lazy Theta*1 keeps the exact same any-angle rule but changes when the check happens. When it generates a successor it optimistically assumes the grandparent is visible (Path 2) and records parent(s2) = parent(s) with g(s2) = g(parent(s)) + euclid without checking. The single LOS query is deferred to the moment s2 is popped for expansion (SetVertex): only then does it verify line_of_sight(parent(s2), s2), and if the optimistic assumption was wrong it repairs the parent to the cheapest already-settled grid neighbour. The result is one LOS check per expanded vertex instead of one per edge — the same any-angle paths, far fewer checks.

In this repository's demo, maze01 settles to Lazy Theta* cost 27.76 · 5 waypoints — essentially the same path as eager Theta* (27.75), produced with a leaner check budget.

How It Works¶

Search on maze01. The frontier grows toward the goal exactly as in Theta*, but the final path is a sparse straight-line polyline grazing only obstacle corners. Where a deferred check rejects the optimistic parent, the vertex is re-parented to a visible grid neighbour before it expands.

Intermediate search progress (left → right: early / middle / final path):

Final result on open01 — with few obstacles, start→goal connects as almost a single straight line:

LAZY-THETASTAR(start, goal):
    g[start] ← 0; parent[start] ← start
    open ← priority queue keyed by f = g + w·h        # h = Euclidean straight-line distance
    while open is not empty:
        s ← open.pop_min()
        if s already settled: continue                # lazy deletion
        SET-VERTEX(s)                                 # deferred LOS check + repair
        settle(s)
        if s == goal: return reconstruct(parent, s)
        for (s2, _) in neighbors(s):
            if s2 not settled:
                UPDATE-VERTEX(s, s2)
    return failure

UPDATE-VERTEX(s, s2):                                 # OPTIMISTIC — no LOS check here
    p ← parent[s]
    cand ← g[p] + euclid(p, s2)                       # assume line_of_sight(p, s2) holds
    if cand < g[s2]:
        g[s2] ← cand; parent[s2] ← p
        open.push(s2, cand + w·h(s2, goal))

SET-VERTEX(s):                                        # run once, when s is popped
    p ← parent[s]
    if p ≠ s and NOT line_of_sight(p, s):             # optimism was wrong → repair
        parent[s] ← argmin_{s' ∈ neighbors(s) ∩ settled} ( g[s'] + euclid(s', s) )
        g[s]      ← min   _{s' ∈ neighbors(s) ∩ settled} ( g[s'] + euclid(s', s) )

The only difference from Theta* is the placement of the line_of_sight call: UPDATE-VERTEX drops it (optimism), and SET-VERTEX adds exactly one at pop time. If the optimistic assumption always holds, no repair ever fires and the returned paths are identical to Theta*; when it fails, the repair falls back to a guaranteed-visible grid neighbour (Path 1 between adjacent cells is always a legal move, and the vertex's generator is such a settled neighbour), so the emitted path is always feasible.

Why the optimism is safe — the settled-neighbour repair¶

The repair set is restricted to already-settled grid neighbours on purpose: their g-values are final, so min(g[s'] + euclid(s', s)) is a real, achievable cost. Because the vertex was generated by some expanded (hence settled) neighbour, that set is never empty — there is always at least one visible fallback, and SET-VERTEX cannot leave a vertex with an invisible parent.

Line of Sight — one collision model with the grid¶

line_of_sight(a, b) decides whether the segment joining two cell centres is traversable using the same corner-cut-forbidden rule as neighbors(). This repository uses the supercover3, which visits every cell the segment touches (delegating to the map's is_motion_valid), so "a pair visible by LOS" ⇔ "a legal straight move," and Lazy Theta* shares one collision model with grid A*. This is inherited unchanged from Theta*; Lazy Theta* only calls it less often.

Heuristic — Euclidean (not octile)¶

Lazy Theta*'s g-values are straight-line (Euclidean) distances, so the heuristic is also Euclidean:

h(a, b) = √((Δrow)² + (Δcol)²)

The octile heuristic the map provides for A* satisfies octile ≥ Euclidean, so it is inadmissible (overestimating) for any-angle costs and is not used. The Euclidean distance is the admissible lower bound for arbitrary-angle motion.

Properties¶

  • Completeness: complete on a finite grid with non-negative costs (same as A* / Theta*).
  • Optimality: like basic Theta*, Lazy Theta* guarantees neither grid-optimality nor true-optimality1 — its parent candidates are still only the (assumed) grandparent or a local grid neighbour, so it can miss the truly shortest any-angle path (though it stays very close).
  • Quality: the returned path is an any-angle straightening of the grid path and its cost tracks eager Theta* closely; the deferred check does not change which shortcuts are taken, only when their visibility is confirmed.
  • Complexity: the same search as A* plus one LOS check per expanded vertex, versus Theta*'s one per relaxation. When LOS checks dominate (long segments, dense grids), this is the practical win.
  • w > 1 (weighted, Pohl 19704): inflating the heuristic reduces expanded nodes at the cost of path quality, exactly as in weighted A*/Theta*.

Lazy vs. Eager — where the checks go¶

Notation. Treat grid cell centres as points in \(\mathbb{R}^2\). An any-angle path \(P=(v_0,\dots,v_k)\) is a polyline of LOS-clear segments with cost \(\operatorname{cost}(P)=\sum_i\lVert v_{i+1}-v_i\rVert_2\), and \(h(n)=\lVert n-\text{goal}\rVert_2\). The admissibility/consistency of \(h\) and the "no longer than grid A*" argument carry over verbatim from Theta*; this section isolates only what the lazy scheme changes.

Lemma (every settled vertex ends with a visible parent). When SET-VERTEX(s) returns, either the optimistic \(\text{parent}(s)=p\) already satisfied LOS\((p,s)\), or the repair reassigned \(\text{parent}(s)=s'\) for some settled grid neighbour \(s'\).

Proof. Let \(s\) be popped and generated (relaxed to its current \(g\)) by a vertex \(u\). Relaxation only happens from a vertex being expanded, so \(u\) is settled, and \(s\) was enumerated in \(\text{neighbors}(u)\), hence \(u\in\text{neighbors}(s)\) (grid adjacency is symmetric). Thus \(\text{neighbors}(s)\cap\text{settled}\neq\varnothing\). Adjacent cells always have LOS (Path 1 is a legal move), so the \(\arg\min\) ranges over a non-empty set of visible candidates and assigns a visible parent. If the optimistic \(p\) was already visible the branch is skipped and the visible \(p\) stays. ∎

Corollary (feasibility). Unrolling the parent chain of any settled \(s\) back to start yields an obstacle-free polyline of length exactly \(g[s]\), since every link \(\text{parent}(x)\to x\) is LOS-clear by the Lemma. In particular the returned \(g[\text{goal}]\) is an achievable upper bound — the path Lazy Theta* emits is always feasible. ∎

Check-count. Theta* evaluates line_of_sight once for every relaxation \((s\to s_2)\); Lazy Theta* evaluates it once per settled vertex inside SET-VERTEX. Since a vertex is settled at most once but relaxed once per incident edge, the number of LOS checks drops from \(\Theta(|E|)\) to \(\Theta(|V|)\) over the explored subgraph. The optimism can occasionally re-parent to a slightly different (equally feasible) polyline, which is why the emitted path is close to but not bit-identical to eager Theta* — the trade is fewer checks for a marginally different any-angle path1.

Parameters¶

Name Type Default Range Description
heuristic_weight float 1.0 [1.0, 5.0] The w in f = g + w·h (h is Euclidean). 1.0 = standard Lazy Theta*, above = weighted

Emitted Trace Events¶

planning_started → (node_expanded, candidate_evaluated, edge_added)* → path_found → planning_finished

No new event type is introduced. Optimistic relaxations emit candidate_evaluated + edge_added with a non-adjacent grandparent as parent (Path 2), exactly as in Theta*. When SET-VERTEX repairs a vertex, it re-emits that vertex with candidate_evaluated + edge_added(state=s, parent=s') where s' is the visible grid neighbour, so replay.py redraws the corrected straight segment and the lazy-repair is visible in the animation. The schema imposes no adjacency constraint on parent.
